reincarnation.util

IntMap<V>

  • V

    The type of values stored in the map

A simple array-based map implementation that uses integer indices as keys. This implementation does not allow null values and treats negative indices as invalid.

IntMap()

Constructs an IntMap with an initial capacity of 8 elements.

get(intindex)V

intindex

The index whose associated value is to be returned
V

The value associated with the specified index, or null if the index is invalid or has no value

Retrieves the value associated with the specified index.

get(intindex, SupplierVdefaultProvider)V

intindex

The index whose associated value is to be returned or computed
SupplierVdefaultProvider

The supplier to generate a default value if none exists
V

The current (existing or computed) value associated with the specified index

Returns the value associated with the specified index, or computes a new value using the provided supplier if no value exists. If a new value is computed, it is stored in the map before being returned.

put(intindex, Vvalue)void

intindex

The index with which the specified value is to be associated
Vvalue

The value to be associated with the specified index (must not be null)

Associates the specified value with the specified index in this map. If the map previously contained a mapping for the index, the old value is replaced. Automatically resizes the internal array if the index exceeds current capacity.

has(intindex)boolean

intindex

The index whose presence in this map is to be tested
boolean

true if this map contains a non-null value for the specified index

Returns true if this map contains a value for the specified index.

delete(intindex)V

intindex

The index whose mapping is to be removed from the map
V

The previous value associated with the specified index, or null if there was no mapping

Removes the mapping for the specified index if present.

find(PredicateVcondition)OptionalV

PredicateVcondition

The predicate to match values against
OptionalV

An Optional containing the first matching value, or empty if none found

Returns an Optional containing the first value that matches the given predicate, or an empty Optional if no value matches. The search is performed in ascending order of indices.
